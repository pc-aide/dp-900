Explanation:

Stream processing is used to handle real-time data and can access recent data or data within a rolling time window. Batch processing typically involves larger volumes of data and does not require immediate processing with low latency. Stream processing can be used for simple calculations like aggregates or rolling averages, but it can also handle more complex tasks in near real-time.

stream processing is often used for simple response functions, aggregates, or calculations such as rolling averages. Stream processing is a type of data processing that allows you to analyze, process and act on data as it is generated or received, in real-time. This type of processing is well-suited for use cases such as monitoring sensor data, analyzing log files, and running real-time analytics on financial or telemetry data, therefore it can be used to perform simple calculations like aggregates or rolling averages. Additionally, it also allows to perform complex calculations and machine learning models in near real-time.
