Correct Answers:

  • Concurrent transactions can't interfere with one another & must result in a consistent database state: Isolation
  • Each Transaction is treated as a single unit that succeeds completely or fails completely: Atomicity
  • Transactions can only take the data in a database from one valid state to another: Consistency
  • When a transaction is committed, it will remain committed: Durability
