Question 13

For each of the following statements, select Yes if the statement is true. Otherwise, select No.

In a transactional database that guarantees atomicity, all statements in a transaction must complete successfully or the transaction must roll back:



In a transactional database that guarantees consistency, all data changes will be logged & accounted for:



In a transactional database that guarantees isolation, processes can see only data in a consistent state:



Explanation:

1 - Atomicity: Atomicity ensures that each transaction is treated as a single unit, which either succeeds completely or fails completely. Therefore, it is not correct to say that all statements in a transaction must complete successfully; the transaction may fail. Thus, the correct answer is No.

2 - Consistency: Consistency ensures that transactions take the database from one valid state to another. Data changes must be logged and accounted for as part of maintaining consistency. Hence, the correct answer is Yes.

3 - Isolation: Isolation guarantees that processes see only data in a consistent state during the transaction. Thus, the correct answer is Yes.

Correct Answers:
In a transactional database that guarantees atomicity, all statements in a transaction must complete successfully or the transaction must roll back: No
In a transactional database that guarantees consistency, all data changes will be logged & accounted for: Yes
In a transactional database that guarantees isolation, processes can see only data in a consistent state: Yes
