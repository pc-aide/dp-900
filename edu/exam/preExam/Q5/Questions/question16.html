Question 16

Which T-SQL statement should be used to instruct a database management system to use an index instead of performing a full table scan?

Explanation:

Select * From Countries where CountryName = "USA"

This is a full table scan because CountryName is not a key; the engine needs to compare every row.

Select * From Countries where CountryID = 33

This is not a full table scan; it becomes an index scan because CountryID is a key, so the engine knows it can skip to that ID.

Select * From Countries

This is a full table scan but with no criteria, so it extracts everything out.

WHERE clause tells a SELECT statement to 'streamline' the query execution based on the search criteria (the PREDICATE) specified in the WHERE clause.

To use an index in a query, if the index is defined on the column specified in the WHERE clause, the query makes use of the specified index:

CREATE TABLE person(
id INT NOT NULL,
first_name VARCHAR(15) NOT NULL,
last_name VARCHAR(15) NOT NULL,
age INT NOT NULL,
PRIMARY KEY(id)
);
/* Index on First Name */
CREATE INDEX person_fname ON person (first_name);

/* This next query shows whether the query uses the intended index */
EXPLAIN SELECT * FROM person WHERE first_name = "Donald";

The correct answer is WHERE, as this is a search condition of the statement and can be used to target one or more indexes. SELECT is incorrect as this doesnâ€™t influence what indexes may or may not be used.
