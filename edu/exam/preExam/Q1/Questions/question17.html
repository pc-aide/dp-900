A key/value data store is optimized for:

Explanation:

A key-value store is the simplest (and often quickest) type of NoSQL database for inserting and querying data. Each data item in a key-value store has two elements, a key and a value. The key uniquely identifies the item, and the value holds the data for the item. The value is opaque to the database management system. Items are stored in key order, making simple lookups extremely efficient.

  • Key/Value Data Stores - Microsoft Docs

Correct Answer:
simple lookups
